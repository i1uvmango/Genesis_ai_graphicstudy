# 🏎️ 자율주행 Behavioral Cloning 프로젝트 설계 명세서

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **목표** | 전문가(MPPI)의 주행 데이터를 학습하여, 시뮬레이션 환경(Genesis)에서 안정적으로 주행하는 제어 모델(MLP) 개발 |
| **프레임워크** | Genesis AI 시뮬레이션 플랫폼 |
| **방법론** | Behavioral Cloning (BC) 기반 지도학습 |

---

## 2. 전문가 데이터 수집 (MPPI Mining)

최고 수준의 전문가 데이터를 생성하기 위해 다음과 같은 고급 제어 기법을 적용함.

### 2.1 Warm-start 전략
- 매 프레임 제어 입력을 새로 계산하지 않고, 이전 스텝의 최적 제어 시퀀스를 **1스텝 Shift**하여 초기 추측값으로 재활용
- 탐색 수렴 속도와 제어 연속성을 비약적으로 향상

### 2.2 Full Solver State Restore
- 단순히 차량의 위치/속도만 동기화하는 것이 아니라, 물리 엔진(Solver)의 **내부 제약 조건과 임펄스 상태 전체**를 저장(`get_state`) 및 복원(`set_state`)
- 시뮬레이션의 물리적 일관성을 완벽히 유지

### 2.3 Colored Noise 탐색
- 가우시안 화이트 노이즈 대신 **AR(1) 구조의 시간 상관 노이즈**를 주입
- 물리적으로 실행 가능한 부드러운 주행 경로를 탐색

## 3. 학습 데이터셋 설계 (DrivingDataset)

데이터의 질을 높이고 모델의 강인함(Robustness)을 확보하기 위한 전략.

### 3.1 입력 벡터 (25D)

| 구분 | 차원 | 내용 |
|------|------|------|
| **현재 상태** | 5D | Longitudinal Velocity ($v_{long}$), Lateral Velocity ($v_{lat}$), Yaw Rate, Cross Track Error (CTE), Heading Error (HE) |
| **미래 정보** | 20D | 미래 10개 지점의 [$v_{long}$, $\kappa$(곡률)] |

### 3.2 Sparse Lookahead 샘플링
- 전문가의 시야(35스텝)와 학생의 입력 차원을 맞추기 위해, **근거리는 촘촘하게, 원거리는 듬성듬성하게** 샘플링
- 샘플링 인덱스: `1, 2, 4, 7, 10, 14, 19, 25, 30, 35`

### 3.3 데이터 증강
- **횡방향 반전(Horizontal Flip)** 을 통해 좌/우 커브 데이터 불균형 해소

### 3.4 DAgger 효과 (노이즈 주입)
- 학습 시 CTE와 HE에 미세 노이즈를 주입
- 모델이 경로를 벗어났을 때 스스로 중앙으로 복귀하는 **복원력(Recovery)** 을 학습

## 4. 모델 아키텍처 및 학습 설정 (ControlMLP)

### 4.1 네트워크 구조

```
입력(25) → Hidden(64, ReLU) → Hidden(64, ReLU) → 출력(2, Tanh)
```

### 4.2 핵심 설계 결정

| 항목 | 내용 | 이유 |
|------|------|------|
| **Tanh 출력 활성화** | 제어 입력 $(T, S)$을 $[-1, 1]$ 범위로 제한 | 분포 외 입력(OOD) 시 발생할 수 있는 속도 폭주 방지 |
| **Weighted MSE Loss** | 조향(Steer) 오차에 가중치 **20배** 부여 | 경로 추종 정밀도 극대화 |
| **단일 StandardScaler** | 훈련/검증 데이터에 동일한 스케일러 적용 | 통계적 일관성 유지 및 데이터 누수(Leakage) 방지 |

---

## 5. 추론 및 실전 주행 (Inference)

### 5.1 Spatial Matching
- 시간 기반 인덱싱 대신, 차량의 현재 위치에서 가장 가까운 경로상의 점을 찾는 **공간 기반 매칭**(`find_nearest_idx`) 적용
- 속도 편차가 발생해도 타겟 경로와의 동기화 유지

### 5.2 Causality 준수
- `drive_any.csv` (이상적 경로): **시각화 가이드라인**으로만 사용
- `golden_inputs.csv` (전문가 데이터): 실제 제어 및 미래 정보 참조에 활용
- → 물리적으로 타당한 인과관계 명확화
