# DAgger (Dataset Aggregation) 기반 반복 학습

## 1. 개요

**DAgger**는 Behavior Cloning(BC)의 **분포 불일치 문제(Distribution Shift)** 를 해결하기 위한 반복적 데이터 수집·학습 방법이다.

BC는 Expert trajectory 데이터로만 학습하기 때문에, 실행 시 모델이 방문하는 상태 분포가 훈련 데이터와 달라지는 문제가 생긴다.

$$p_{\text{expert}}(s) \neq p_{\text{policy}}(s)$$

이를 **error cascade** 라 하며, 작은 오차가 점점 누적되어 결국 실패로 이어진다.

```
차선 중앙 → 약간 오른쪽 → 더 오른쪽 → 차선 이탈
```

> DAgger는 모델이 실제로 방문한 상태에서의 Expert 행동을 데이터셋에 추가하며 반복 학습하여 이 문제를 해결한다.

---

## 2. BC vs DAgger 비교

| 항목 | Behavior Cloning | DAgger |
| :--- | :--- | :--- |
| 데이터 분포 | Expert trajectory만 | Expert + Policy trajectory |
| OOD 대응 | ❌ 불가능 | ✅ 가능 |
| Recovery 행동 | 학습 불가 | 학습 가능 |
| 학습 방식 | 1회 학습 | 반복 학습 |

---

## 3. 전체 파이프라인

```
① Expert Dataset 생성  (MPPI로 golden T, S 산출)
② 초기 BC 학습
③ Policy Rollout       (실제 환경에서 실행)
④ 실패 상태 수집
⑤ Expert Label 생성    (실패 구간에 MPPI 재실행)
⑥ Dataset Aggregation  (D ← D ∪ {(s, a_expert)})
⑦ 모델 재학습
⑧ ③~⑦ 반복
```

### 각 단계 설명

| 단계 | 내용 |
| :--- | :--- |
| **① Expert Dataset** | MPPI 등 Expert planner로 `(s_t, a*_t)` 쌍 생성 → `D₀` |
| **② 초기 BC 학습** | `π₀ = argmin E[(a - a*)²]` |
| **③ Policy Rollout** | 학습된 policy를 시뮬레이션에서 실행, 방문 상태 기록 |
| **④ 상태 수집** | OOD 발생 구간(차선 이탈 직전, 급커브 진입 등) 수집 |
| **⑤ Expert Label** | 수집한 상태 `s`에 MPPI를 재실행하여 `a_expert` 산출 |
| **⑥ Aggregation** | `D ← D ∪ {(s, a_expert)}` 로 dataset 확장 |
| **⑦ 재학습** | 확장된 dataset으로 `π₁` 재학습 |

---

## 4. 수집해야 할 데이터 (Recoverable States)

DAgger에서 핵심은 **복구 가능한 실패 상태(Recoverable Failure State)** 를 수집하는 것이다.

✅ **포함해야 할 상태 (예시)**
- 차선 중앙에서 0.5m 이탈
- Yaw error 소폭 발생
- 커브 진입 직전

❌ **제외해야 할 상태**

| 제외 유형 | 설명 |
| :--- | :--- |
| 물리적 불가능 상태 | 충돌, 차량 전복 → episode reset |
| 극단적 발산 상태 | lane deviation > 1.5m, yaw error > 30° |
| Terminal failure | 완전한 도로 이탈 → episode 종료 |

---

## 5. Recovery 데이터가 중요한 이유

BC의 error cascade 문제:

```
0.5° error → 2° error → lane departure  (BC는 회복 불가)
```

Recovery 데이터가 있으면 모델이 다음을 학습한다:

```
차선 오른쪽 치우침 → 왼쪽 steering
차선 왼쪽 치우침  → 오른쪽 steering
```

즉 **복구 정책(Recovery Policy)** 이 생기므로 error cascade를 방지할 수 있다.
