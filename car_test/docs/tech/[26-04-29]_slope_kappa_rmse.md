# Sim2Sim MPPI 평가 구간 분류 기준 및 근거

## 개요

Genesis MPPI 마이닝 결과를 정량 평가할 때,  
경로 전체를 단순 평균(RMSE)으로 평가하면 **어려운 구간의 실패가 희석**된다.  
이를 해결하기 위해 경로를 **물리적 난이도 기준으로 구간 분류**하고,  
구간별로 가중치를 달리 적용한 **weighted RMSE**를 사용한다.

---

## 1. 종단경사 (Slope) 분류 기준

### 출처
- 국토교통부 「도로의 구조·시설 기준에 관한 규칙」
- AASHTO Green Book (A Policy on Geometric Design of Highways and Streets)

### 기준값

| 등급 | 조건 | 실제 도로 기준 | 의미 |
|------|------|----------------|------|
| **flat** | \|slope\| < 0.03 | 평지부 (3% 미만) | 중력 저항 무시 가능 |
| **mild** | 0.03 ≤ \|slope\| < 0.09 | 구릉지~산지부 (3~9%) | 구동력의 일부가 경사 저항으로 소모 |
| **steep** | \|slope\| ≥ 0.09 | 특수구간 (9% 초과) | 일반 국도 최대허용경사(9%) 초과 |

### 물리적 근거

```
경사 저항력 = M × g × sin(θ) ≈ M × g × slope

flat  (3%):  저항력 = 차량중량의 3%   → 구동력 대비 무시 가능
mild  (9%):  저항력 = 차량중량의 9%   → 명확한 속도 감소 발생
steep (15%): 저항력 = 차량중량의 15%  → 저출력 차량은 등판 불가
```

## 2. 평면곡률 (Kappa, κ) 분류 기준

### 출처
- RBC 차량 동역학 스케일 역산 (설계속도 ~10km/h 기준)
- 최소곡선반경 공식: R_min = V² / (127 × (e + f))

### 기준값

| 등급 | 조건 | 곡선반경 환산 | 의미 |
|------|------|---------------|------|
| **straight** | \|κ\| < 0.1 | R > 10m | 조향각 변화 미미, 횡력 무시 가능 |
| **curve** | 0.1 ≤ \|κ\| < 0.3 | R = 3~10m | 명확한 조향 필요, 횡력 발생 |
| **sharp** | \|κ\| ≥ 0.3 | R < 3m | 차체 길이 대비 반경 6배 이내, 언더스티어 급증 |

### 스케일 선택 근거

일반 도로 기준(설계속도 60km/h)을 그대로 적용하면  
RBC 차량(~10km/h)의 모든 커브가 "직선"으로 분류된다.  
→ 설계속도 비례로 κ 임계값을 **6배 상향** 조정하여 RBC 스케일에 맞춤.

## 3. Transition 구간 (전환점)

### 정의

종단경사(slope)의 **부호가 바뀌는 지점** 전후 **±10프레임**을 별도 구간으로 분류.

```
Crest (볼록형): 오르막 → 내리막
  frame: ... +slope ... [transition] ... -slope ...
                              ↑
                         pitch 각속도 최대

Sag (오목형): 내리막 → 오르막
  frame: ... -slope ... [transition] ... +slope ...
                              ↑
                         서스펜션 최대 압축
```

### ±10프레임 근거

| 항목 | 값 |
|------|-----|
| dt (시뮬레이션 타임스텝) | 0.04167 s |
| 10프레임 = | 0.417 s |
| 차량 pitch 고유진동수 (추정) | 1~2 Hz |
| 반주기 | 0.25~0.5 s |

10프레임(0.417s)은 pitch 반주기를 포함하는 최소 윈도우.  
좁으면(±3프레임) 실제 pitch 발산 구간을 놓치고,  
넓으면(±15프레임) transition이 인접 구간을 과도하게 잠식한다.  
**±10프레임은 두 조건을 절충한 실용적 선택.**

### 우선순위

Transition은 slope/kappa 분류보다 **최우선** 적용된다.

```
분류 우선순위:
  1순위: transition  (slope 부호 변화점 ±10프레임)
  2순위: steep_sharp, steep_curve 등 복합구간
  3순위: 나머지 slope/kappa 조합
```

## 4. 복합구간 가중치 근거

### 출처
- FHWA Highway Design Handbook
- 합성경사(Resultant Grade) 개념

### 합성경사 공식

```
경사와 곡률이 동시에 작용할 때 실제 차량이 받는 등가 경사:

e_resultant = sqrt(slope² + (B × κ)²)
  B: 차량 폭/2

단순 합산이 아닌 벡터 합성 → 복합구간이 단독보다 항상 더 어렵다
```

### 가중치 설계 원칙

```
기준: flat_straight = 1.0

slope 단독 증가:  ×1.5~2.0  (경사 저항만 추가)
kappa 단독 증가:  ×1.5~2.0  (횡력만 추가)
복합 구간:        두 배율의 곱  (두 힘이 동시 작용)

예시:
  mild(×1.5) × curve(×1.5) = 2.25 → mild_curve = 2.5  ✅
  steep(×2.0) × sharp(×2.5) = 5.0 → steep_sharp = 5.0 ✅
```

### 가중치 테이블

|                  | straight (×1.0) | curve (×1.5) | sharp (×2.0) |
|------------------|-----------------|--------------|--------------|
| **flat (×1.0)**  | 1.0             | 1.5          | 2.0          |
| **mild (×1.5)**  | 1.5             | 2.5          | 3.5          |
| **steep (×2.0)** | 2.0             | 3.5          | 5.0          |

| transition | 4.0 (steep_curve와 steep_sharp 사이) |
|------------|--------------------------------------|

## 5. 평가 지표

### 구간별 계산 항목

| 지표 | 수식 | 의미 |
|------|------|------|
| RMSE_XY | √(mean((Δx²+Δy²))) | XY 경로 추종 오차 |
| RMSE_Z | √(mean(Δz²)) | 높이 추종 오차 |
| RMSE_Pitch | √(mean(Δpitch²)) | 차체 자세 오차 |

### 최종 Score

```
weighted_score = Σ(weight_i × RMSE_XY_i) / Σ(weight_i)
  i: 존재하는 구간 (n_frames ≥ 5인 구간만)

→ 낮을수록 좋음
→ Optuna find_params.py의 최적화 target
```

### RMSE 단독 사용의 한계와 보완

| 한계 | 보완 방법 |
|------|-----------|
| 경사 구간 실패가 평지 구간에 희석됨 | 구간별 가중 RMSE 사용 |
| 전체 평균이 낮아도 특정 구간 완전 실패 가능 | 구간별 개별 출력으로 확인 |
| lag 발생 시 frame 비교가 부정확 | lag를 별도 지표로 추적 (mean_lag) |
| 완주 실패를 RMSE로 표현 불가 | completion_rate + rollback_count 병행 |

## 6. 한계 및 주의사항

```
1. RBC 스케일 미검증
   κ 임계값은 역산 추정치 → 실험으로 보정 필요

2. slope 계산 노이즈
   slope = gradient(z) / gradient(s)
   terrain 보간 오차가 slope에 전파될 수 있음
   → preprocess_df의 Gaussian smoothing 후 분류 적용됨

3. transition ±10프레임은 실용적 선택
   P08(Up_then_Down) 경로에서 pitch_sim vs pitch_ref 플롯으로
   실제 발산 구간 폭을 확인하여 조정 가능

4. n_frames < 5 구간 제외
   통계적으로 의미없는 구간은 weighted_score 계산에서 제외
   → 짧은 경로에서 일부 구간이 아예 없을 수 있음
```
